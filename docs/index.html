Three ways to build
the same pipeline

Bank Pulse ingests FDIC financial data on every insured U.S. institution and transforms it through a Bronze → Silver → Gold medallion architecture — implemented three different ways so each approach can be directly compared.

Source: FDIC BankFind Suite API · ~27,000 institutions · quarterly since 1992

Medallion Architecture

Bronze
bronze_financials
Raw JSON from FDIC API · 50k records · ingested_at, batch_id metadata
Silver
silver_financials
Typed columns · deduplicated by cert + report_date · DQ flags
Gold
gold_bank_metrics
NIM · NPL ratio · peer group · QoQ changes per institution

Ingestion Approaches

⚡

Databricks Jobs

Notebook-based orchestration

Three PySpark notebooks wired together as sequential job tasks. Each layer reads from and writes to Delta tables explicitly, with full control over partitioning and schema management.

  • Task DAG declared with depends_on in job YAML
  • Delta Lake writes via .write.saveAsTable()
  • DQ tracked as boolean flag columns in silver layer
  • GIT source — notebooks pulled from GitHub at runtime
PySpark Delta Lake DAB
🔷

Databricks DLT Pipeline

Delta Live Tables · declarative

The same Bronze → Silver → Gold logic rewritten using DLT decorators. The pipeline infers execution order from dlt.read() references and tracks data quality natively in the pipeline event log.

  • DAG auto-inferred from dlt.read() references
  • DLT manages persistence — no explicit writes
  • DQ via @dlt.expect — metrics in event log
  • Triggered or continuous mode via continuous: false
DLT PySpark DAB
❄️

dbt + Snowflake

SQL-first transformations

Bronze data loaded into Snowflake via internal stage and COPY INTO. dbt builds a silver view and a gold table using pure SQL, with built-in testing and schema documentation.

  • Bronze loaded by load_snowflake.py via COPY INTO
  • stg_financials view — cleans and deduplicates
  • gold_bank_metrics table — NIM, peer group, QoQ
  • dbt tests enforce not-null constraints on key columns
Snowflake dbt SQL
